## 배운 점

### 배열 마지막 인덱스 (Chapter05 Problem06)

잘못된 코드

```java
scores[scores.length]
```

올바른 코드

```java
scores[scores.length - 1]
```

배열의 마지막 인덱스는 length가 아니라 length - 1 이다.

---

## 느낀 점

반복문을 이용해 다양한 조건을 처리하는 방법에 익숙해졌다.

배열에서는 length를 활용한 순회와 최대·최소값 찾기, 역순 출력 방법을 배울 수 있었다.

특히 배열 인덱스가 0부터 시작하고 마지막 인덱스는 length - 1이라는 점을 실수하면서 확실하게 이해할 수 있었다.

## 배운 점

### 두 번째로 큰 수 찾기 (Chapter05 Problem11)

두 번째로 큰 수를 찾을 때는 최대값만 구하는 것보다 한 단계 더 복잡한 사고가 필요했다.

최대값의 위치를 저장한 뒤, 해당 위치를 제외하고 다시 가장 큰 값을 찾는 방식으로 해결했다.

배열 문제에서는 값뿐만 아니라 인덱스를 함께 관리하는 것이 중요하다는 점을 배울 수 있었다.

---

### 최빈값 찾기 (Chapter05 Problem16)

최빈값 문제는 Chapter05에서 가장 어려운 문제였다.

각 숫자가 몇 번 등장했는지 별도의 배열에 저장한 뒤, 가장 많이 등장한 값을 찾는 방식으로 해결했다.

또한 동률일 때 먼저 나온 숫자를 유지하기 위해 `>`를 사용하고 `>=`를 사용하지 않는다는 점을 배울 수 있었다.

---

### 연속 구간 관리 (Chapter05 Problem23)

가장 긴 연속 문자 찾기 문제는 문자열 문제 중 가장 오래 고민한 문제였다.

처음에는 for문 하나로 구현하려고 했지만, 연속된 문자가 끊어졌을 때 count를 다시 1로 초기화해야 한다는 점을 놓쳐 구현에 어려움을 겪었다.

이 문제를 통해 현재 연속 길이(seqCount), 최대 연속 길이(maxSeq), 최대 길이의 문자(maxCh)처럼 여러 상태 변수를 동시에 관리하는 방법을 연습할 수 있었다.

또한 문제를 해결한 뒤에는 while문과 for문을 함께 사용하는 방식뿐만 아니라, for문 하나만으로도 같은 로직을 더 간단하게 구현할 수 있다는 점을 이해했다.
